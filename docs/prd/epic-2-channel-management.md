# Epic 2: 频道管理命令

## Epic目标
实现全面的频道生命周期管理命令，允许用户通过CLI创建、列出、查看、更新和删除直播频道。

## 用户故事
作为用户（内部运营人员或PaaS客户开发者），我想通过命令行界面管理直播频道，以便我能够高效地处理频道操作而无需使用Web控制台。

## 前置条件
- Epic 1（CLI基础与认证）必须完成

## 故事列表

### 故事 2.1: 频道创建命令
- **目标**: 实现 `channel create` 命令来创建新的直播频道
- **需求**:
  - 接受必需参数：`--name <string>`
  - 接受可选参数：`--description <string>`, `--max-viewers <number>`, `--auto-record <boolean>`
  - 验证输入参数
  - 调用PolyV API创建频道
  - 以表格格式显示成功/错误结果
- **验收标准**:
  1. 命令接受 `--name` 参数（必需）
  2. 命令接受可选参数，具有适当的默认值
  3. 参数验证显示清晰的错误消息
  4. 成功创建时以表格格式显示频道ID、名称和状态
  5. API错误以用户友好的消息优雅处理
  6. 命令支持 `--output json` 进行机器可读输出

### 故事 2.2: 频道列表命令  
- **目标**: 实现 `channel list` 命令来显示现有频道并支持分页
- **需求**:
  - 接受可选参数：`--page <number>`, `--limit <number>`
  - 默认分页：page=1, limit=20
  - 调用PolyV API获取频道列表
  - 以表格格式显示结果，包含ID、名称、创建时间、状态
  - 支持JSON输出格式
- **验收标准**:
  1. 命令使用默认分页列出频道
  2. 分页参数正常工作
  3. 表格输出格式良好且可读
  4. JSON输出有效且完整
  5. 优雅处理空结果
  6. API错误显示适当消息

### 故事 2.3: 频道获取命令
- **目标**: 实现 `channel get` 命令来获取详细的频道信息
- **需求**:
  - 接受必需参数：`--channelId <ID>`
  - 验证频道ID格式
  - 调用PolyV API获取频道详情
  - 显示全面的频道信息
  - 支持表格和JSON输出格式
- **验收标准**:
  1. 命令需要 `--channelId` 参数
  2. 频道ID验证防止无效输入
  3. 详细频道信息清晰显示
  4. 表格格式人类可读
  5. JSON格式包含所有可用数据
  6. 不存在的频道显示适当错误消息

### 故事 2.4: 频道更新命令
- **目标**: 实现 `channel update` 命令来修改现有频道属性
- **需求**:
  - 接受必需参数：`--channelId <ID>`
  - 接受可选更新参数：`--name <string>`, `--description <string>`
  - 验证所有输入参数
  - 调用PolyV API更新频道
  - 显示更新确认
- **验收标准**:
  1. 命令需要 `--channelId` 参数
  2. 必须提供至少一个更新参数
  3. 所有字段的参数验证都正常工作
  4. 成功更新显示确认消息
  5. API错误得到适当处理
  6. 找不到频道显示清晰错误消息

### 故事 2.5: 频道删除命令
- **目标**: 实现带有安全确认机制的 `channel delete` 命令
- **需求**:
  - 接受必需参数：`--channelId <ID>`
  - 实现安全确认（交互式或 `--force` 标志）
  - 调用PolyV API删除频道
  - 显示删除确认
  - 优雅处理删除错误
- **验收标准**:
  1. 命令需要 `--channelId` 参数
  2. 交互式确认提示用户进行验证
  3. `--force` 标志绕过交互式确认
  4. 成功删除显示确认消息
  5. API错误以清晰消息处理
  6. 不存在的频道显示适当错误

## 技术注意事项
- 所有命令必须使用Epic 1的认证系统
- 所有命令必须支持 `--output table|json` 参数
- API调用应包含适当的错误处理和重试
- 表格格式应使用cli-table3库
- 命令验证应在API调用之前进行
