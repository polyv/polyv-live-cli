# Epic 3: 流控制命令

## Epic目标
实现直播流控制命令，允许用户管理流操作，包括获取流密钥、开始和停止直播。

## 用户故事
作为用户（内部运营人员或PaaS客户开发者），我想通过命令行界面控制直播操作，以便我能够管理流生命周期并以编程方式获取流凭据。

## 前置条件
- Epic 1（CLI基础与认证）必须完成
- Epic 2（频道管理命令）必须完成

## 故事列表

### 故事 3.1: 流获取密钥命令
- **目标**: 实现 `stream get-key` 命令来获取RTMP URL和流密钥
- **需求**:
  - 接受必需参数：`--channelId <ID>`
  - 验证频道ID格式
  - 调用PolyV API获取流凭据
  - 清晰显示RTMP URL和流密钥
  - 支持表格和JSON输出格式
  - 处理敏感数据显示的安全考虑
- **验收标准**:
  1. 命令需要 `--channelId` 参数
  2. 频道ID验证防止无效输入
  3. RTMP URL和流密钥清晰显示
  4. 表格格式人类可读且安全
  5. JSON格式包含所有流凭据
  6. 不存在的频道显示适当错误消息
  7. 敏感数据处理遵循安全最佳实践

### 故事 3.2: 流开始命令
- **目标**: 实现 `stream start` 命令来为频道开始直播
- **需求**:

  - 验证频道ID和频道状态
  - 调用PolyV API开始流
  - 显示开始确认和状态
  - 优雅处理已开始的流
  - 为失败提供清晰的错误消息
- **验收标准**:
  1. 命令需要 `--channelId` 参数
  2. 频道ID验证正常工作
  3. 成功开始显示确认消息
  4. 已开始的流显示信息消息
  5. API错误以清晰解释处理
  6. 找不到频道显示适当错误
  7. 流状态清晰传达

### 故事 3.3: 流停止命令
- **目标**: 实现 `stream stop` 命令来结束频道的直播
- **需求**:

  - 显示停止确认和最终状态
  - 适当处理已停止的流

  4. 已停止的流显示信息消息
  5. API错误以用户友好的消息处理
  6. 找不到频道显示适当错误
  7. 最终流状态清晰显示

## 技术注意事项
- 所有命令必须使用Epic 1的认证系统
- 流凭据（RTMP URL、密钥）应安全处理
- 命令应在流操作前验证频道存在
- 错误消息应区分频道错误和流错误
- 考虑流操作的速率限制和API节流
- 流操作可能需要时间 - 提供适当的用户反馈

## API集成注意事项
- 流获取密钥：获取频道的流凭据
- 流开始：启动直播会话
- 流停止：终止活动流会话
- 所有操作需要适当的频道ID和认证
- 应验证API响应的完整性
