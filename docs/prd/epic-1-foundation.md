# Epic 1: CLI基础与认证

## Epic目标
建立核心CLI基础设施，具备强大的认证、配置管理和基本命令结构，以实现与PolyV API的安全通信。

## 用户故事
作为开发者或运营人员，我想要一个安全可靠的CLI基础，处理认证并提供一致的命令行界面，以便我能够安全高效地管理PolyV直播服务。

## 故事列表

### 故事 1.1: 项目设置和基础CLI结构
- **目标**: 创建基础的TypeScript CLI项目结构，使用Commander.js框架
- **需求**:
  - 初始化具有适当package.json的TypeScript/Node.js项目
  - 为命令行界面设置Commander.js
  - 配置构建系统（TypeScript编译）
  - 设置基础错误处理框架
  - 创建主入口点（`src/index.ts`）
- **验收标准**:
  1. CLI可以通过 `npm run build` 构建
  2. 构建后CLI二进制文件可执行
  3. 基础帮助命令（`--help`）显示使用信息
  4. 版本命令（`--version`）显示当前版本
  5. 错误处理提供用户友好的消息

### 故事 1.2: 认证系统实现
- **目标**: 使用appId、appSecret和可选userId实现安全认证
- **需求**:
  - 支持命令行参数：`--appId`, `--appSecret`, `--userId`
  - 支持环境变量：`POLYV_APP_ID`, `POLYV_APP_SECRET`, `POLYV_USER_ID`
  - 命令行参数优先级高于环境变量
  - 安全存储和处理凭据
- **验收标准**:
  1. CLI通过命令行参数接受认证
  2. CLI通过环境变量接受认证
  3. 命令行参数覆盖环境变量
  4. 缺少必需凭据时显示清晰错误消息
  5. 凭据不会在错误消息中被记录或暴露

### 故事 1.3: MD5签名实现
- **目标**: 实现PolyV的基于MD5的签名机制以确保API安全
- **需求**:
  - 实现参数排序和拼接
  - 使用appSecret三明治模式生成MD5哈希
  - 处理时间戳生成（毫秒）
  - 过滤null/undefined/空参数
  - 创建可重用的签名工具
- **验收标准**:
  1. 签名正确遵循PolyV的算法规范
  2. 参数按键名字母顺序排序
  3. 时间戳以毫秒生成
  4. 过滤null/undefined/空参数
  5. 签名格式正确（大写MD5）
  6. 签名工具经过单元测试

### 故事 1.4: 基础配置管理
- **目标**: 实现配置加载和验证系统
- **需求**:
  - 支持使用dotenv加载.env文件
  - 验证必需的配置参数
  - 为缺少的配置提供清晰的错误消息
  - 支持不同环境（dev, prod）
- **验收标准**:
  1. 配置从.env文件加载
  2. 缺少必需配置时显示有用的错误消息
  3. 配置验证防止无效值
  4. 不同环境可以使用不同的配置文件
  5. 配置是类型安全的（TypeScript接口）

## 依赖关系
- 无（这是基础epic）

## 技术注意事项
- 使用TypeScript以获得类型安全和更好的开发体验
- Commander.js作为CLI框架（稳定、文档完善）
- Axios用于HTTP客户端（将在后续epic中使用）
- crypto模块用于MD5签名生成
- dotenv用于环境变量管理
