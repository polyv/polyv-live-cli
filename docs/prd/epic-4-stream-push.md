# Epic 4: 发起推流命令

## Epic目标
实现一个客户端推流命令，允许用户使用本地视频文件向指定的PolyV频道发起直播推流。这为用户提供了一个完整的、端到端的CLI工作流程，从创建频道到发起直播。

## 用户故事
作为一名用户（开发者或运营人员），我希望能通过CLI将一个本地视频文件推送到指定的频道，以便我能快速测试直播流或进行简单的直播，而无需依赖复杂的第三方推流软件。

## 前置条件
- Epic 1 (CLI基础与认证) 必须完成。
- Epic 2 (频道管理命令) 必须完成。
- Epic 3 (流控制命令) 必须完成，特别是 `stream get-key` 的功能，因为推流需要获取RTMP推流地址和密钥。
- 用户的本地环境需要安装 `ffmpeg`。

## 故事列表

### 故事 4.1: 实现 `stream push` 命令
- **目标**: 创建一个新的 `stream push` 命令，用于将本地视频文件推送到指定频道的RTMP服务器。
- **需求**:
  - 命令接受两个必需参数: `--channelId <ID>` 和 `--file <path>`。
  - 命令在执行前，应首先检查用户的环境中是否安装了 `ffmpeg`。如果未安装，应给出清晰的提示。
  - 命令内部需要调用 `stream get-key` 的逻辑，来获取指定频道的RTMP推流地址和密钥。
  - 命令会启动一个 `ffmpeg` 子进程来执行推流操作。
  - `ffmpeg` 的输出（标准输出和标准错误）应该被捕获并实时显示给用户，以便用户了解推流状态和诊断问题。
  - 用户应该能够通过 `Ctrl+C` 来平滑地中止推流进程。
- **验收标准**:
  1. 当不提供 `--channelId` 或 `--file` 参数时，命令会报错并提示用法。
  2. 如果系统中找不到 `ffmpeg`，命令会打印错误信息并退出。
  3. 如果提供的文件路径不存在，命令会打印错误信息并退出。
  4. 命令能成功获取RTMP地址和密钥，并正确构建 `ffmpeg` 命令。
  5. `ffmpeg` 进程成功启动，并将视频流推送到PolyV服务器。
  6. 用户可以在PolyV的播放器中观看到正在直播的视频流。
  7. 用户可以通过 `Ctrl+C` 正常停止推流。

## 技术注意事项
- **外部依赖**: 此功能核心依赖于`ffmpeg`。需要设计一种健壮的方式来检测`ffmpeg`是否存在。
- **子进程管理**: 需要正确地创建和管理`ffmpeg`子进程，包括处理它的生命周期、输入输出流以及终止信号。
- **安全性**: 拼接 `ffmpeg` 命令时，必须谨慎处理来自用户输入的文件路径和从API获取的参数，以防范命令注入风险。
- **用户反馈**: `ffmpeg`的输出信息可能非常冗长。可以考虑进行适当的格式化或过滤，只向用户展示关键信息（如帧率、比特率、错误等）。

## API集成注意事项
- 此Epic主要依赖于Epic 3中实现的 `get_stream_info` API调用，以获取推流所需的 `rtmpUrl` 和 `streamKey`。没有新的服务端API集成。
