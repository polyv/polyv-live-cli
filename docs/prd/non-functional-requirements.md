# 非功能性需求与项目范围

## 1. 非功能性需求 (Non-Functional Requirements)

- **1.1. 性能**: 单个命令的执行响应时间（不含网络延迟）应在200ms以内。
- **1.2. 易用性**:
    - 必须提供全局的`-h` / `--help`参数，以及针对每个子命令的帮助信息。
    - 帮助信息应清晰易懂，包含参数说明和使用示例。
    - 错误信息必须对用户友好，能清晰地指出问题所在（如"错误：缺少必要的参数'--name'"）。
- **1.3. 兼容性**: 需支持主流的操作系统：Linux, macOS, 和 Windows (via PowerShell/WSL)。
- **1.4. 可靠性**: 对OpenAPI的调用失败（如网络超时、API返回500错误）需要有重试机制或明确的失败提示。

---

## 2. MVP范围之外 (Out of Scope)

- 所有与显式场次管理相关的功能。
- 除基础状态外的复杂实时数据监控。
- 录播文件的管理。
- 直播间互动功能（签到、抽奖等）的管理。

---

## 3. 衡量指标 (Success Metrics)

- **采纳率**: 发布6个月内，至少20%的活跃PaaS客户在自动化脚本中集成CLI。
- **效率提升**: 内部团队批量任务的平均耗时降低50%。
- **用户满意度**: 通过收集GitHub Issue和社区反馈，用户正面评价占比超过80%。
