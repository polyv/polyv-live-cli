# 故事 1.1: 项目设置和基础CLI结构

## 状态: 完成

## 故事

- 作为一个开发者
- 我想要设置基础的TypeScript CLI项目和Commander.js框架
- 以便我能够在一个坚实、可扩展的基础上构建PolyV CLI命令

## 验收标准 (ACs)

1. CLI项目可以通过 `npm run build` 成功构建
2. 构建后CLI二进制文件可执行
3. 基础帮助命令 (`--help`) 显示全面的使用信息
4. 版本命令 (`--version`) 显示package.json中的当前版本
5. 错误处理框架为常见场景提供用户友好的消息
6. 项目遵循TypeScript最佳实践，具有适当的类型定义
7. Commander.js正确集成用于命令解析

## 开发注意事项

### 前一个故事的洞察
- 这是第一个故事，没有前面故事的上下文

### 技术栈要求
[来源: architecture/overview.md#2]
- **编程语言**: TypeScript，用于类型安全和可维护性
- **运行时**: Node.js LTS版本，用于稳定性和生态系统支持
- **CLI框架**: Commander.js，作为最稳定和流行的CLI框架
- **包管理**: npm，用于分发和依赖管理

### 项目结构指导原则
[来源: architecture/system-design.md#1]
- **主入口**: `src/index.ts` - CLI入口点，用于注册所有命令
- **分层架构**: 表示层（命令）→ 业务逻辑 → 服务层 → 工具
- **源目录**: 所有TypeScript源代码应在 `src/` 目录中
- **构建输出**: 编译的JavaScript应输出到 `dist/` 目录

### 包配置要求
[来源: architecture.md#5.3]
- **包名**: `polyv-live-cli` 
- **二进制命令**: `polyv-cli` (可执行命令名)
- **主文件**: `dist/index.js` (编译入口点)
- **文件字段**: 发布包中只包含 `/dist` 目录
- **关键脚本**: 
  - `build`: TypeScript编译 (`tsc`)
  - `start`: 执行编译的CLI (`node dist/index.js`)
  - `test`: 测试运行器 (`jest`)

### 构建配置详情
[来源: architecture.md#5.2]
- **TypeScript构建**: 使用 `tsc` 命令进行编译
- **Node.js版本**: 针对Node.js LTS版本以确保兼容性
- **分发**: 包必须可通过 `npm install -g polyv-live-cli` 安装
- **二进制设置**: 正确的bin字段配置用于全局CLI安装

### 文件位置
- 主入口点: `src/index.ts`
- 包配置: `package.json` (项目根目录)
- TypeScript配置: `tsconfig.json` (项目根目录)
- 构建输出: `dist/` 目录
- 工具模块: `src/utils/` 目录

### 技术约束
- 必须兼容主要操作系统（Linux, macOS, Windows）
- 应遵循Node.js CLI最佳实践进行全局安装
- TypeScript配置应启用严格类型检查
- 错误处理应保持一致和用户友好
