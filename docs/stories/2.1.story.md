# 故事 2.1: 频道创建命令

## 状态: 完成

## 故事

- 作为一个用户（内部运营人员或PaaS客户开发者）
- 我想要实现 `channel create` 命令来创建新的直播频道
- 以便我能够通过CLI高效地管理频道创建，而无需使用Web控制台

## 验收标准 (ACs)

1. 命令接受 `--name` 参数（必需）
2. 命令接受可选参数，具有适当的默认值
3. 参数验证显示清晰的错误消息
4. 成功创建时以表格格式显示频道ID、名称和状态
5. API错误以用户友好的消息优雅处理
6. 命令支持 `--output json` 进行机器可读输出

### 频道创建API要求
[来源: docs/api/channel/create.md]
- **端点**: POST `/live/v4/channel/create`
- **认证**: 在URL参数中使用appId、timestamp、sign的MD5签名
- **内容类型**: application/json
- **必需的主体参数**: name、newScene、template
- **可选的主体参数**: channelPasswd、pureRtcEnabled、linkMicLimit、categoryId、startTime、endTime
- **响应格式**: 带有成功/错误结构的V4 API格式

### 数据模型要求
[来源: docs/architecture/data-models.md]
- **ChannelModel接口**: 带验证规则的完整频道表示
- **认证模型**: 用于API请求的MD5签名生成
- **API响应模型**: 带错误处理的V4 API响应包装器
- **验证规则**: 名称最大100字符，channelPasswd 6-16位字母数字，场景/模板枚举

### 技术实现要求
[来源: docs/architecture/tech-stack.md]
- **HTTP客户端**: 使用带请求/响应拦截器的Axios 1.6.0
- **CLI框架**: 用于命令注册的Commander.js 11.1.0
- **表格输出**: 用于格式化控制台输出的cli-table3 0.6.3
- **TypeScript**: 带显式类型和接口的严格模式
- **测试**: 带全面单元和集成测试的Jest 29.7.0

### 文件结构要求
[来源: docs/architecture/source-tree.md]
- **命令**: `src/commands/channel.commands.ts` 用于CLI命令定义
- **处理器**: `src/handlers/channel.handler.ts` 用于业务逻辑编排
- **服务**: `src/services/channel.service.ts` 用于API集成
- **类型**: `src/types/channel.ts` 用于TypeScript类型定义
- **测试**: 与.test.ts后缀的同位置单元测试，tests/目录中的集成测试

### 集成要求
[来源: docs/architecture/external-apis.md]
- **签名生成**: 使用故事1.3的现有MD5签名工具
- **配置**: 与故事1.4的配置系统集成以获取凭据
- **错误处理**: 使用故事1.1错误框架的PolyVError类
- **输出格式**: 支持表格和JSON输出格式

### API端点详情
[来源: docs/api/channel/create.md]
- **URL**: `http://api.polyv.net/live/v4/channel/create`
- **方法**: 带JSON主体的POST
- **URL参数**: appId、timestamp、sign（用于认证）
- **主体参数**: name（必需）、newScene、template（必需），加上可选参数
- **响应**: V4格式，在数据字段中包含channelId、userId、channelPasswd

### 验证和错误处理
[来源: docs/architecture/coding-standards.md, docs/architecture/data-models.md]
- **输入验证**: 验证名称长度、场景/模板枚举、密码格式
- **错误消息**: 验证失败和API错误的用户友好消息
- **类型安全**: 对所有请求/响应数据使用TypeScript接口
- **错误分类**: 区分验证、认证和API错误
