# Story 9-2: 回放详情命令

## Status: done

## 故事

**作为一个** 运营人员或 PaaS 宷户开发者，
**我想要** 通过 CLI 使用 channelId 和 videoId 获取特定回放视频的详细信息，
**以便** 我能够快速查看单个回放视频的完整配置信息。

## 验收标准

1. **AC1**: `playback get` 命令通过 `--channel-id` 和 `--video-id` 参数获取指定回放视频的详情
2. **AC2**: 返回完整的回放信息（包含视频ID、标题、时长、状态、创建时间等)
3. **AC3**: 表格输出格式清晰，显示视频ID、标题、时长、创建时间、状态等关键信息
4. **AC4**: JSON 输出完整包含所有字段
5. **AC5**: 指定的回放视频不存在时显示友好的错误提示
6. **AC6**: 支持 `--list-type` 参数区分回放列表(playback)和点播列表(vod)

7. **AC7**: 鑽鏈复用已有的 `PlaybackHandler` 类,扩展 `getPlayback` 方法

8. **AC8**: 遵循 ATDD 开发模式,先编写测试,再实现功能

## Dev Notes
### 技术栈要求
[来源: docs/architecture/tech-stack.md]
- **TypeScript**: 5.3.3 - 使用严格类型检查，所有组件必须有明确类型
- **Node.js**: 20.11.0 LTS - 运行时环境
- **Commander.js**: 11.1.0 - CLI框架用于命令注册
- **Jest**: 29.7.0 - 测试框架
### API 集成要求
- **API端点**: 使用现有的 `GET /live/v2/channel/recordFile/{channelId}/playback/list` API
- **认证**: MD5签名认证 (appId, timestamp, sign)
- **SDK服务**: 使用已有的 `ChannelService.getPlaybackList()` 方法
- **过滤逻辑**: 根据 videoId 从列表中筛选指定回放
### 编码标准要求
[来源: docs/architecture/coding-standards.md]
- 所有 public 方法必须有 JSDoc 注释
- 使用明确的 TypeScript 类型,避免 any 类型
- 异步操作必须有完整的错误处理
- 遵循现有的命名约定 (camelCase变量, PascalCase类)
- 单元测试覆盖率必须 >= 80%
### 错误处理模式
- 回放视频不存在时返回友好的错误消息
- 继承现有的错误处理模式
- API错误响应应转换为用户友好的消息
### 命令集成模式
[来源: 现有 playback.commands.ts 模式]
- 命令注册函数: 夲 `registerPlaybackGetCommand(program: Command)` 添加到现有命令
- 选项解析: 使用Commander.js的option方法
- 错误包装: 使用 `wrapCommand` 装饰器
- 帮助文本: 遵循现有命令的帮助格式
