# Story 4.1: 实现 `stream push` 命令

## 用户故事
作为一名用户（开发者或运营人员），我希望能通过CLI将一个本地视频文件推送到指定的频道，以便我能快速测试直播流或进行简单的直播，而无需依赖复杂的第三方推流软件。

## 技术实施细节

### 1. **命令定义与参数**
- **命令**: `stream push`
- **参数**:
  - `--channelId <ID>`: (必需) 目标直播频道的ID。
  - `--file <path>`: (必需) 本地视频文件的路径。

### 2. **前置检查**
- **`ffmpeg` 依赖检查**:
  - 在执行任何操作前，程序必须检查 `ffmpeg` 是否在系统的 `PATH` 中可用。
  - 可以通过执行 `ffmpeg -version` 并检查其退出码来实现。如果退出码不为0，则表示`ffmpeg`不可用。
  - 若 `ffmpeg` 不存在，应向用户显示一条清晰的错误消息，并指引他们安装，然后程序应中止。
- **文件存在性检查**:
  - 验证 `--file` 参数提供的路径是否指向一个真实存在的文件。
  - 如果文件不存在，显示错误消息并退出。

### 3. **核心逻辑**
- **获取推流凭证**:
  - 复用或调用 `stream.handler.ts` 中已有的逻辑来执行 `get-key` 操作。
  - 这将返回频道的 `rtmpUrl` 和 `streamKey`。
  - 必须处理获取凭证过程中可能发生的错误（例如，频道不存在、API请求失败等）。
- **构建 `ffmpeg` 命令**:
  - 基于获取到的凭证和用户提供的文件路径，构建 `ffmpeg` 命令。
  - 命令模板: `ffmpeg -re -i "<file_path>" -c:v copy -c:a aac -f flv "<rtmp_url>/<stream_key>"`
  - **参数解释**:
    - `-re`: 以视频的原始帧率读取输入，模拟实时流。
    - `-i "<file_path>"`: 指定输入文件。路径应使用引号包裹，以处理包含空格的路径。
    - `-c:v copy`: 直接复制视频流，不进行重新编码，以节省CPU资源。
    - `-c:a aac`: 将音频流编码为AAC格式，这是RTMP协议广泛支持的格式。
    - `-f flv`: 指定输出格式为FLV，这是RTMP协议的标准容器格式。
    - `"<rtmp_url>/<stream_key>"`: 拼接完整的RTMP推流地址。同样，整个URL应使用引号包裹。
- **执行推流**:
  - 使用Node.js的 `child_process` 模块来异步执行 `ffmpeg` 命令。
  - 必须将 `ffmpeg` 进程的 `stdout` 和 `stderr` 管道输送到主进程，并实时展示给用户。这对于监控推流状态和调试错误至关重要。

### 4. **进程管理与用户交互**
- **优雅地终止**:
  - 监听 `SIGINT` 信号（当用户按下 `Ctrl+C` 时触发）。
  - 收到信号后，应向 `ffmpeg` 子进程发送终止信号，并等待其完全退出。
  - 向用户显示一条消息，确认推流已停止。
- **错误处理**:
  - 监控 `ffmpeg` 进程的 `error` 和 `exit` 事件。
  - 如果 `ffmpeg` 因错误退出（例如，无效的文件格式、网络中断），应捕获其错误输出，并以清晰、易于理解的方式呈现给用户。

## 验收标准
1. 当不提供 `--channelId` 或 `--file` 参数时，命令会报错并提示用法。
2. 如果系统中找不到 `ffmpeg`，命令会打印错误信息并退出。
3. 如果提供的文件路径不存在，命令会打印错误信息并退出。
4. 命令能成功获取RTMP地址和密钥，并正确构建 `ffmpeg` 命令。
5. `ffmpeg` 进程成功启动，并将视频流推送到PolyV服务器。
6. 用户可以在PolyV的播放器中观看到正在直播的视频流。
7. 用户可以通过 `Ctrl+C` 正常停止推流。
