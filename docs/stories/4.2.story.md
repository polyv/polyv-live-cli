# Story 4.2: 实现流验证功能增强

## 用户故事
作为一名用户（开发者或运营人员），我希望能在推流过程中获得实时的流验证信息和性能监控，以便我能确保直播质量、及时发现问题并获得完整的流传输报告，从而提高直播的可靠性和用户体验。

## 背景与依赖
- 依赖 Story 4.1 的 `stream push` 命令实现
- 基于现有的 `stream status` 和 `getStreamStatus` 功能
- 需要增强现有的状态监控能力

## 功能需求

### 1. 增强的流推送验证 (`stream push --verify`)
- **命令**: `stream push --channelId <ID> --file <path> --verify`
- **新增参数**:
  - `--verify`: 启用流验证模式
  - `--verification-interval <seconds>`: 验证检查间隔 (默认: 30秒)
  - `--quality-threshold <fps>`: 质量阈值，低于此FPS时发出警告 (默认: 15)
  - `--show-viewer-links`: 显示观看链接

### 2. 独立的流验证命令 (`stream verify`)
- **命令**: `stream verify --channelId <ID>`
- **参数**:
  - `--channelId <ID>`: (必需) 目标频道ID
  - `--duration <seconds>`: 验证持续时间 (默认: 60秒)
  - `--interval <seconds>`: 检查间隔 (默认: 10秒)
  - `--output <format>`: 输出格式 (table/json，默认: table)
  - `--save-report <path>`: 保存验证报告到文件

### 3. 实时状态监控 (`stream monitor`)
- **命令**: `stream monitor --channelId <ID>`
- **参数**:
  - `--channelId <ID>`: (必需) 目标频道ID
  - `--refresh <seconds>`: 刷新间隔 (默认: 5秒)
  - `--alerts`: 启用质量警告
  - `--output <format>`: 输出格式 (table/json，默认: table)

## 技术实施细节

### 1. 流验证核心逻辑
```typescript
interface StreamVerificationOptions {
  channelId: string;
  duration?: number;
  interval?: number;
  qualityThreshold?: number;
  showViewerLinks?: boolean;
  saveReport?: string;
}

interface StreamVerificationResult {
  channelId: string;
  verificationId: string;
  startTime: Date;
  endTime: Date;
  totalChecks: number;
  successfulChecks: number;
  failedChecks: number;
  averageMetrics: {
    fps: number;
    bandwidth: number;
    latency?: number;
  };
  qualityIssues: QualityIssue[];
  viewerLinks: ViewerLink[];
  summary: VerificationSummary;
}
```

### 2. 推流验证增强
- 在现有 `pushStream` 方法中集成验证逻辑
- 启动独立的验证线程进行周期性检查
- 实时显示验证状态和性能指标
- 在发现质量问题时提供警告

### 3. 质量监控指标
- **基础指标**: FPS、带宽、延迟
- **质量阈值**: 可配置的FPS、带宽下限
- **状态监控**: 流状态变化检测
- **错误检测**: 连接中断、编码错误等

### 4. 观看链接生成
- 获取频道的播放地址
- 支持多种播放协议 (HLS, RTMP, WebRTC)
- 生成测试观看链接

### 5. 验证报告生成
```typescript
interface VerificationReport {
  metadata: {
    channelId: string;
    testDuration: number;
    testInterval: number;
    timestamp: Date;
  };
  summary: {
    overallStatus: 'excellent' | 'good' | 'fair' | 'poor';
    reliability: number; // 0-100%
    averageQuality: QualityMetrics;
  };
  timeline: VerificationPoint[];
  issues: QualityIssue[];
  recommendations: string[];
}
```
