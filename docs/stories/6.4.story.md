# 故事 6.4: 配置文件安全性

## 状态: ✅ COMPLETED (企业级安全实现)

## 故事

**作为一个** 使用PolyV CLI进行账号管理的开发者或运营人员，
**我想要** 账号配置文件能够安全地存储和访问，包含适当的加密和权限控制，
**以便** 我的应用密钥和敏感信息得到妥善保护，避免泄露风险，同时支持配置的可靠备份和恢复。

## 验收标准

### 原始验收标准 ✅ 全部完成
1. ✅ **appSecret在配置文件中被加密存储**: 使用AES-256-GCM企业级加密算法保护敏感信息
2. ✅ **配置文件权限设置为600（仅用户可读写）**: 限制其他用户访问配置文件，跨平台支持
3. ✅ **支持 `POLYV_MASTER_KEY` 环境变量作为加密密钥**: 允许用户提供自定义加密密钥
4. ✅ **配置文件损坏时显示恢复指导**: 提供清晰的错误提示和修复步骤
5. ✅ **配置文件格式版本化，支持未来迁移**: 包含版本信息以支持向后兼容
6. ✅ **加密/解密失败时有清晰的错误提示**: 提供具体的故障排除指导

### 实际实现的增强功能（超出原始范围）
7. ✅ **多层安全架构**: 实现了完整的安全管理系统
   - `AccountEncryption`: AES-256-GCM加密系统
   - `AccountCrypto`: 加密工具类
   - `SecureAccountManager`: 集成安全管理器
8. ✅ **密钥管理系统**: 支持自定义密钥、默认密钥生成、密钥验证
9. ✅ **完整性验证**: 加密数据完整性检查、数字签名验证
10. ✅ **安全错误处理**: 不在错误消息中泄露敏感信息的安全错误处理
11. ✅ **跨平台权限管理**: Windows ACL和Unix文件权限的统一管理

### 加密实现设计
[来源: 技术分析和安全最佳实践]
- **AccountEncryption接口**:
  ```typescript
  interface AccountEncryption {
    encrypt(plaintext: string): EncryptedData;
    decrypt(encryptedData: EncryptedData): string;
    generateKey(): string;
    validateKey(key: string): boolean;
  }
  
  interface EncryptedData {
    algorithm: 'aes-256-gcm';
    iv: string;           // 初始化向量（base64编码）
    authTag: string;      // 认证标签（base64编码）
    encrypted: string;    // 加密数据（base64编码）
  }
  ```
- **密钥生成策略**: 结合用户环境信息生成确定性但安全的默认密钥
- **认证加密**: 使用GCM模式确保数据完整性和认证性

### 配置文件结构更新
[来源: docs/architecture/data-models.md和安全需求]
- **安全化的accounts.json格式**:
  ```typescript
  interface SecureAccountsConfig {
    version: string;                    // 配置文件格式版本（如"1.0"）
    accounts: {
      [accountName: string]: {
        appId: string;                  // 明文存储
        appSecret: EncryptedData;       // 加密存储
        userId?: string;                // 明文存储（可选）
        createdAt: string;              // 创建时间
        updatedAt: string;              // 更新时间
      };
    };
    metadata: {
      createdAt: string;
      lastBackup?: string;              // 最后备份时间（可选）
      keySource: 'environment' | 'generated'; // 密钥来源
    };
  }
  ```

### 安全约束和最佳实践
[来源: docs/architecture/external-apis.md#安全考虑]
- **密钥管理**:
  - appSecret加密存储，不以明文保存
  - 加密密钥不与配置文件存储在同一位置
  - 支持密钥轮换和更新机制
- **文件安全**:
  - 配置文件权限限制为仅当前用户访问
  - 定期检查权限设置，自动修复不安全配置
  - 支持配置文件的安全备份
- **错误处理安全**:
  - 错误消息中不暴露敏感信息
  - 提供安全的诊断信息
  - 区分系统错误和用户错误

### 跨平台兼容性
[来源: docs/architecture/tech-stack.md#操作系统支持]
- **Linux/macOS**: 使用chmod系统调用设置文件权限
- **Windows**: 使用Node.js fs模块的权限API或Windows ACL
- **权限检查**: 实现跨平台的权限验证逻辑
- **错误处理**: 处理不同操作系统的权限错误

### 数据模型扩展

  interface SecurityContext {

  }

### 错误处理策略
[来源: docs/architecture/coding-standards.md#错误处理标准]
- **加密错误**: 使用 `PolyVSecurityError` 类型
- **权限错误**: 使用 `PolyVPermissionError` 类型
- **配置错误**: 使用 `PolyVConfigError` 类型
- **详细错误信息**: 包含具体的安全问题和建议的解决步骤
