# 故事 6.1: 账号配置管理

## 状态: ✅ COMPLETED (超出原始范围)

## 故事

**作为一个** 管理多个PolyV客户账号的开发者或运营人员，
**我想要** 能够添加、删除、列出和查看多个PolyV账号配置，
**以便** 我可以便捷地管理多个账号信息，为后续的会话切换功能做好基础准备。

## 验收标准

### 原始验收标准 ✅ 全部完成
1. ✅ **可以成功添加新的账号配置**: 使用 `polyv-cli account add <name> --app-id <id> --app-secret <secret> [--user-id <uid>]` 命令
2. ✅ **账号名称重复时显示清晰错误消息**: 提供用户友好的错误提示和建议
3. ✅ **appSecret被安全加密存储在配置文件中**: 使用AES-256-GCM加密，敏感信息不以明文保存
4. ✅ **可以列出所有已配置的账号（不显示敏感信息）**: 显示账号名称、appId等非敏感信息
5. ✅ **可以删除指定的账号配置**: 支持安全的账号删除操作，支持`--force`选项
6. ✅ **删除账号时需要确认操作**: 避免意外删除重要配置
7. ✅ **配置文件存储在 `~/.polyv/accounts.json`**: 使用标准配置路径

### 额外实现的功能（超出原始范围）
8. ✅ **遗留配置迁移**: `polyv-cli account migrate [--name <name>] [--force] [--keep-legacy]`
9. ✅ **默认账号管理**: `polyv-cli account set-default <name>` 和 `account unset-default`
10. ✅ **增强的当前状态显示**: 显示会话账号、默认账号、认证来源等详细信息
11. ✅ **多种输出格式**: 支持 `--output json|table` 选项
12. ✅ **中文用户界面**: 提供中文界面和错误提示

### 多账号配置架构
[来源: docs/prd/epic-6-multi-account-management.md]
- **配置文件位置**: `~/.polyv/accounts.json`
- **加密策略**: 使用Node.js内置crypto模块，AES-256-GCM算法
- **配置格式**:
  ```typescript
  interface AccountsStore {
    version: string;                    // 配置文件版本
    accounts: Record<string, AccountConfig>; // 账号名称 -> 配置映射
    metadata: {
      createdAt: string;
      updatedAt: string;
    };
  }
  
  interface AccountConfig {
    name: string;                       // 账号名称（唯一标识）
    appId: string;                      // PolyV应用ID（明文）
    appSecret: string;                  // 加密后的PolyV应用密钥
    userId?: string;                    // 可选的用户ID
    createdAt: string;                  // 创建时间
    updatedAt: string;                  // 更新时间
  }
  ```

### 加密和安全要求
[来源: docs/prd/epic-6-multi-account-management.md#故事-6.4]
- **加密算法**: AES-256-GCM，提供认证加密
- **密钥管理**: 支持环境变量 `POLYV_MASTER_KEY`，如未设置则使用默认派生密钥
- **文件权限**: 配置文件设置为600权限（仅用户可读写）
- **敏感信息处理**: appSecret不得出现在日志、错误消息或控制台输出中

### 命令行接口设计
[来源: docs/prd/epic-6-multi-account-management.md]
- **添加账号**: `polyv-cli account add <name> --app-id <id> --app-secret <secret> [--user-id <uid>]`
- **删除账号**: `polyv-cli account remove <name>`
- **列出账号**: `polyv-cli account list [--output table|json]`
- **显示当前**: `polyv-cli account current`

[来源: docs/architecture/data-models.md]
- **配置验证**: 
  ```typescript
  const AccountConfigValidation = {
    name: {
      required: true,
      pattern: /^[a-zA-Z0-9_-]+$/,  // 字母数字下划线横线
      maxLength: 50
    },
    appId: {
      required: true,
      pattern: /^[a-zA-Z0-9]+$/,    // 字母数字
      minLength: 8,
      maxLength: 32
    },
    appSecret: {
      required: true,
      minLength: 16,                 // 最小16字符
      maxLength: 128                 // 最大128字符
    }
  };
  ```
