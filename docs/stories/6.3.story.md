# 故事 6.3: 认证优先级重构

## 状态: ✅ COMPLETED (实现了5级认证系统)

## 故事

**作为一个** 使用多种认证方式（命令行参数、会话账号、环境变量、config配置）的PolyV CLI用户，
**我想要** 系统能够按照明确的优先级顺序选择认证信息，并保持与现有config命令的完全兼容，
**以便** 我可以灵活地在不同场景下使用最合适的认证方式，同时确保现有配置和工作流程不受影响。

## 验收标准

### 原始验收标准 ✅ 全部完成
1. ✅ **命令行参数具有最高优先级**: `--app-id`, `--app-secret`, `--user-id` 参数始终覆盖其他认证源
2. ✅ **会话账号在没有命令行参数时生效**: `polyv-cli use <name>` 设置的当前账号作为第二优先级
3. ✅ **环境变量作为第三优先级**: `POLYV_APP_ID`, `POLYV_APP_SECRET`, `POLYV_USER_ID` 在没有更高优先级时使用
4. ✅ **全局config配置作为最后的默认配置**: `polyv-cli config` 设置的配置作为最低优先级
5. ✅ **现有config命令的所有功能完全兼容**: 未使用account功能的用户体验完全不变
6. ✅ **未使用account功能的用户体验完全不变**: 单账号用户继续正常使用config命令
7. ✅ **认证失败时明确指出使用的认证来源**: 错误消息包含当前使用的认证方式和来源
8. ✅ **认证成功时可选显示当前账号来源信息**: 在verbose模式下显示当前使用的认证来源

### 实际实现的增强功能（超出原始4级系统）
9. ✅ **5级认证优先级系统**: 在第3和第4级之间添加了默认账号层级
   - 优先级1: 命令行参数
   - 优先级2: 会话账号 (`polyv-cli use <name>`)
   - 优先级3: 环境变量
   - **新增** 优先级4: 默认账号 (`polyv-cli account set-default <name>`)
   - 优先级5: 全局config配置

10. ✅ **默认账号系统**: `polyv-cli account set-default <name>` 和 `account unset-default`
11. ✅ **认证源详细追踪**: 包含账号名称、优先级、描述等详细信息
12. ✅ **中文认证信息显示**: 本地化的认证来源显示和错误提示

### 认证优先级系统架构
[来源: docs/prd/epic-6-multi-account-management.md#故事-6.3]
- **4级认证优先级体系**:
  1. **命令行参数**（最高优先级）：`--app-id`, `--app-secret`, `--user-id`
  2. **会话账号**：`polyv-cli use <account-name>` 设置的当前账号
  3. **环境变量**：`POLYV_APP_ID`, `POLYV_APP_SECRET`, `POLYV_USER_ID`
  4. **全局config配置**（最低优先级）：`polyv-cli config` 设置的默认配置
- **向后兼容性**: 现有config命令的所有功能完全不受影响
- **认证来源追踪**: 记录每次认证使用的来源和元数据

### 认证源管理设计
[来源: docs/architecture/external-apis.md和技术分析]
- **AuthenticationSource接口**:
  ```typescript
  interface AuthenticationSource {
    type: 'command-line' | 'session' | 'environment' | 'config';
    priority: number;           // 1=最高，4=最低
    appId?: string;
    appSecret?: string;
    userId?: string;
    metadata: {
      source: string;           // 具体来源描述
      accountName?: string;     // 账号名称（如适用）
      setAt?: Date;            // 设置时间
    };
  }
  ```
- **认证源检测策略**: 按优先级顺序检测，找到第一个完整的认证信息即停止
- **认证信息完整性验证**: 至少需要appId和appSecret，userId可选

### PolyV API认证要求
[来源: docs/architecture/external-apis.md#认证机制]
- **MD5签名认证**: 所有API请求需要MD5签名
- **必需参数**: appId, timestamp, sign
- **可选参数**: userId（流控制操作需要）
- **签名生成**: appSecret + 参数拼接 + appSecret，MD5加密转大写
- **时间戳要求**: 13位毫秒级时间戳，3分钟有效期
