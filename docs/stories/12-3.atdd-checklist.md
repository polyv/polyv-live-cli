
### CLI Commands

```bash
# watch-condition get
watch-condition get [--channel-id <channelId>] [-o table|json]

# watch-condition set (simple)
watch-condition set [--channel-id <channelId>] --rank <1|2>
                    --auth-type <type> --enabled <Y|N>
                    [--code <password>] [--price <amount>]
                    [-o table|json]

# watch-condition set (from config file)
watch-condition set [--channel-id <channelId>] --config <filepath>
                    [-o table|json]
```

## Notes

- Follow the pattern from `ViewerHandler` and `viewer.commands.ts`
- Reuse SDK WebService methods (getWatchCondition, setWatchCondition)
- All public methods must have JSDoc comments
- Table output should display rank, enabled, authType, and type-specific details
- CLI short option rule: **NO `-v` or `-V` allowed** (conflicts with `--version`)
- Business rule: When primary condition is disabled, secondary cannot be enabled
- Business rule: Primary and secondary cannot have the same authType when both enabled
