## 四、发现的Bug和修复

### Bug #1: `-v`短选项与全局`--version`冲突

**问题描述**:
`viewer get`命令的`-v`短选项与CLI全局的`-v, --version`选项冲突，导致命令无法正常执行。

**复现步骤**:
```bash
polyv-live-cli viewer get -v "viewer_id" -a nicksu
# 输出: 1.2.17 (版本号，而非执行命令)
```

**修复方案**:
将`-v`改为`-i`（代表id），避免与全局选项冲突。

**修复代码**:
```typescript
// 修改前
.requiredOption('-v, --viewer-id <id>', 'viewer unique ID')

// 修改后
.requiredOption('-i, --viewer-id <id>', 'viewer unique ID')
```

**影响范围**:
- `packages/cli/src/commands/viewer.commands.ts`
- `packages/cli/src/commands/viewer.commands.test.ts`
- 帮助文档示例

**验证**:
修复后重新构建并测试，命令正常工作。
