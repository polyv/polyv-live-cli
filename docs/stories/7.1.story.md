# 故事 7.1: 实现产品列表和获取命令

## 状态: Ready for Review

## 故事

**作为一个** 使用PolyV CLI管理直播频道产品的运营人员或开发者，
**我想要** 通过命令行界面查看频道的产品列表和获取特定产品详情，
**以便** 我能够快速了解频道下的所有产品配置，进行产品管理和运营决策。

## 验收标准

1. **产品列表命令**: 实现 `product list` 命令支持分页参数 (page, size)
2. **输出格式支持**: 支持现有的输出格式 (table/json)，具有合适的列格式化
3. **过滤和排序功能**: 包含与现有频道列表命令一致的过滤和排序功能
4. **API集成**: 正确调用 `GET /live/v3/channel/product/list` API并处理分页功能
5. **错误处理**: 实现完整的错误处理，包括认证失败、频道不存在等情况
6. **输出一致性**: 表格输出与现有CLI命令保持一致的视觉风格

### 数据模型要求
[来源: docs/architecture/data-models.md]
产品数据模型应包含：
```typescript
interface ProductModel {
  productId: string;                // 产品ID
  channelId: string;                // 关联的频道ID
  name: string;                     // 产品名称
  productType: 'normal' | 'finance' | 'position'; // 产品类型
  status: number;                   // 产品状态
  linkType: number;                 // 链接类型
  cover?: string;                   // 产品封面图
  price?: number;                   // 价格
  realPrice?: number;               // 实际价格
  createdTime: number;              // 创建时间
  updatedTime: number;              // 更新时间
}
```

- **错误处理**: 统一的PolyV错误响应格式
