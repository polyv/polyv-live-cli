# 故事 6.2: 会话级账号切换

## 状态: ✅ COMPLETED (超出原始范围)

## 故事

**作为一个** 需要在不同终端中使用不同PolyV账号的开发者或运营人员，
**我想要** 能够在每个终端会话中独立切换和使用不同的PolyV账号，
**以便** 我可以在多个终端窗口中同时操作不同的账号，而无需每次都手动输入账号参数，提高工作效率并避免账号混用的风险。

## 验收标准

### 原始验收标准 ✅ 全部完成
1. ✅ **可以成功切换当前终端的账号**: 使用 `polyv-cli use <account-name>` 命令
2. ✅ **不同终端可以同时使用不同的账号**: 会话状态互不干扰
3. ✅ **当前账号状态在终端会话中保持**: 重新执行命令时使用相同账号
4. ✅ **切换到不存在的账号时显示错误消息**: 提供清晰的错误提示和建议
5. ✅ **`polyv-cli account current` 显示当前会话使用的账号**: 明确显示会话账号信息
6. ✅ **未设置账号时运行命令显示友好提示**: 指导用户如何设置账号

### 额外实现的功能（超出原始范围）
7. ✅ **会话清除功能**: `polyv-cli use --clear` 清除当前会话账号设置
8. ✅ **会话状态查询**: `polyv-cli use --status` 显示详细的当前会话状态
9. ✅ **可用账号列表**: `polyv-cli use --list` 列出所有可配置的账号
10. ✅ **会话清理功能**: `polyv-cli use --cleanup` 清理过期的会话文件
11. ✅ **会话过期管理**: 支持会话过期时间和自动清理
12. ✅ **进程和终端ID跟踪**: 精确的会话隔离和状态追踪
13. ✅ **中文用户界面**: 完全本地化的中文界面

### 会话状态管理架构
[来源: docs/prd/epic-6-multi-account-management.md#故事-6.2]
- **存储策略**: 使用环境变量和临时文件结合的方式存储会话状态
- **会话隔离**: 每个终端进程独立维护账号状态
- **状态格式**:
  ```typescript
  interface SessionState {
    accountName: string;           // 当前使用的账号名称
    processId: number;             // 关联的进程ID
    terminalId: string;            // 终端标识符
    setAt: Date;                   // 设置时间
    expiresAt?: Date;              // 可选过期时间
  }
  ```

### 会话存储机制设计
[来源: docs/architecture/tech-stack.md和项目需求分析]
- **主存储**: 环境变量 `POLYV_SESSION_ACCOUNT` 存储当前会话账号名
- **辅助存储**: 临时目录下的会话文件作为备份验证
- **进程隔离**: 每个终端进程有独立的环境变量空间
- **清理机制**: 会话状态随终端关闭自动清除
- **跨平台路径**: 
  - Linux/macOS: `~/.polyv/sessions/`
  - Windows: `%USERPROFILE%\.polyv\sessions\`

### 命令行接口设计
[来源: docs/prd/epic-6-multi-account-management.md]
- **切换账号**: `polyv-cli use <account-name>`
- **显示当前**: `polyv-cli account current` (扩展显示会话信息)
- **清除会话**: `polyv-cli use --clear` (清除当前会话设置)
